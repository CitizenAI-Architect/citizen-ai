# CUCI Logic: The Hybrid Anchor & Personal Fog

## 1. The Fog Engine (Under the Hood)
The mathematical algorithm responsible for data fragmentation and encryption. 
- **Operation:** Works invisibly "under the hood" to ensure seamless user experience.
- **Personalized Fog:** During first activation, the engine generates a unique "Mathematical Salt" (Personal Seed). This ensures that even with the same code, every user produces a unique "fog signature".

## 2. The Hybrid Key (The Lens Trigger)
The mechanism that activates the Fog Engine to "unfog" the data. It consists of two parts:

- **Static Part (The "Dead" Key):** An encrypted file-based backup. It can be stored anywhere (cloud, friends, external drive) as it is useless without the activation pulse.
- **Dynamic Part (The "Live" Impulse):** - **Fast Access:** Biometric activation for daily local use.
    - **Sovereign Access:** A passphrase or code stored only in the user's mind. Essential for recovery on new devices.

## 3. Security Principles
1. **Zero-Knowledge:** The system never knows the user's passphrase or the contents of the "unfogged" data.
2. **Device Independence:** The CUCI can be reconstructed on any clean device using the Static Backup and the Sovereign Access code.
3. **Collision Resistance:** Personal Seeds prevent different users' data from overlapping or being decrypted by the same logic.
